
一、檔案位置﹕

標題檔案位在區域目錄下﹐如以洛陽為例﹐洛陽目錄下的檔案與目錄如下﹕

  [lc] mud:~/mud/area/loyang$ ls
  index  mob/   obj/   res/   roo/   shp/

其中 index 檔案就是標題檔案﹐為區域所必須﹐若缺乏則區域無法讀入。

二、格式﹕

*             /* 若有 * 開頭的這一行則為註解﹐系統不會讀入 */
Echo          <字串常數或數字>
Editor        <字串>~
Name          <字串>~
Flags         <字串常數或數字>
Upper         <字串常數或數字>
Lower         <字串常數或數字>
Fog           <數字>
Serial        <數字>
Capital       <數字>
Description
<字串>
~

End           /* 標示標題檔案的結束 */

三、範例﹕

Echo          NO_ECHO
Editor        Cutty~
Name          特殊地方~
Upper         LEVEL_PC
Lower         1
Flags         SECTION_NEWHAND
Fog           0
Serial        0
Capital       0
Description
這是一個特殊地方﹐你看不見我﹐除非你是神﹐嘿嘿﹗
~

End

四、說明﹕

  (1) Echo (區域的顯現屬性)﹕

    NO_ECHO                0    不顯現
    WILL_ECHO              1    會顯現

    1. 不設定時預設值為 1, 會顯現。
    2. 區域若設定為不顯現﹐則只有神族可以看的見。

  (2) Editor ( 區域撰寫人 )﹕
    1. 一定要設定, 否則系統不能啟動。
    2. 最後面一定要有字串結束符號( ~ )。
    3. 可以有多人以上﹐代表此區域有多人撰寫。

  (3) Name ( 區域名稱 )﹕
    1. 一定要設定, 否則系統不能啟動。
    2. 後面要有字串結束符號( ~ )。

  (4) Upper (建議較高等級)﹕

    LEVEL_GOD               120
    LEVEL_SUP               119
    LEVEL_DEI               118
    LEVEL_ANG               117
    LEVEL_HER               116
    LEVEL_IMM               101
    LEVEL_PC                100

    1. 此為你建議可以逛此區域的等級﹐若高於此等級不會有太多
       的經驗值。
    2. 可以用字串代替數字。
    3. 在撰寫區域時除了城池之外﹐請將怪物等級限定在一個範圍
       內﹐讓玩家可以有固定的地方升級。

  (5) Lower (建議較低等級)﹕

    字串常數同上項 Upper 所提到。

    1. 此為你所建議逛此區域的最低等級﹐若低於此等級將無法勝任。
    2. 可以用字串代替數字。

  (6) Flags (區域旗標)﹕

    SECTION_FLY              1        空中區域
    SECTION_SWIM             2        水中區域
    SECTION_NEWHAND          4        新手區

    1. 如果設定旗標則區域所有的房間都會依照此設定。
    2. 如果房間設定中有相對於此處的設定﹐則以房間為準。
    3. 若設定為新手區﹐則玩家無法攻擊比其等級高六級的怪物。

  (7) Fog (迷霧指數)﹕
    1. 代表這個區域出現大霧的機率。
    2. 數值從 0 到 100。
    3. 區域若出現大霧，玩家看到的房間敘述上會有部份字元以虛線表示。

  (8) Serial (區域序號)
    1. 區域序號為區域房間號碼以百單位為計算。
    2. 例如區域房間號碼若為 501 - 551 則序號為 5。

  (9) Capital (出生地房間)
    1. 若有設定這個房間號碼，則本區域會自動設定成出生城。
    2. 出生城為新玩家註冊時可以做為籍貫的選擇。
    3. 設定的房間必須存在，否則無法啟動。
    4. 設定的房間則為玩家使用 recall 傳送回出生地時所到的房間。
    5. 房間號碼設定為 0 時代表本區域不是出生城。

  (10) Description (區域描述)﹕
    1. 此為區域的輔助說明﹐在 area help <區域號碼> 時可以見到。
    2. 若為城池建議說明下列項目﹕
      a. 位置  ﹕ 說明此區域的位置。
      b. 交通  ﹕ 說明此區域如何與其他地方溝通。
      c. 怪物  ﹕ 說明怪物等級的分佈。
      d. 轉職者﹕ 說明城內有哪一些轉職者供玩家轉職。
      e. 訓練者﹕ 說明城內有哪一些訓練者供玩家訓練屬性。
    3. 若非城池則非常希望能說明如何到達﹐解迷可以不用述說。

  (8) End ( 代表此格式結束的符號 )﹕
    如果結束﹐一定要有 End。
